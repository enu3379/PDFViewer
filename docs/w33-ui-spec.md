## 0. 용어 결정 (D2 선행 결정)

| 내부 용어 | 사용자 언어 | 이유 |
|---|---|---|
| sync / syncing | **연동 / 연동중** | "싱크"는 외래어 + 클라우드 오해 소지. 로컬 개념임을 "연동"이 더 잘 전달 |
| undecided | **미결정** | 정본 용어 그대로 — 중립적, "끊김"같은 부정 뉘앙스 없음 |
| detached | (배지 없음) | 독립 문서가 기본값이므로 이름을 노출하지 않는다 — 연동 해제 메뉴에서만 "연동 해제" 동사로 등장 |
| syncHub | **원본** (문구에서만) | "허브"는 개발 용어. 힌트 문구에선 "○○의 사본입니다"로 관계만 말한다 |

## 1. D1 — 상태 배지 (점·알약)

기존 알약 칩 규격(높이 18px, 폰트 11px, radius 999px)과 마진 점(6px) 재사용. **pen 테마(클래식/소다)와 무관하게 항상 앰버 계열** — 주석 색과 혼동 방지.

| 상태 | 점 | 알약 | 텍스트 | 노출 |
|---|---|---|---|---|
| 연동중 | 채운 원 `--accent` | 배경 `--accent-wash`, 테두리 없음 | `--accent-deep` "연동중" | 뷰어 툴바(문서명 옆) + 허브 카드 |
| 미결정 | 빈 원, 1.4px 테두리 `--soft` | 투명 배경, **1.4px dashed** `--line` | `--muted` "미결정" | 뷰어 툴바 + 허브 카드 |
| 해제/독립 | — | — | — | **배지 없음** (대부분의 문서가 독립 — 소음 방지) |

- 허브 카드에서는 알약 대신 **점만** 카드 우상단에 표시(카드 밀도 유지), hover 시 툴팁으로 상태명.
- 뷰어 툴바에서는 점+텍스트 알약. 클릭 = 연동 액션 트리거(§3 플로우 F2).

수용 기준: 클래식·소다 테마 모두에서 배지가 하이라이트 4색과 시각적으로 구분된다. 독립 문서는 어떤 배지도 렌더링하지 않는다.

## 2. D1 — 아이콘 (16×16, stroke 1.5, `currentColor`, fill none)

기존 저장 버튼을 **분할 버튼**으로 확장: 기본 클릭 = 파일만 저장(현행 동작 유지), 우측 캐럿 = 메뉴.

```html
<!-- 파일만 저장: 문서 + 아래 화살표 -->
<svg viewBox="0 0 16 16"><path d="M4 1.5h5.5L13 5v9.5H4z"/><path d="M9.5 1.5V5H13"/><path d="M8.5 7.5v4M6.7 9.7l1.8 1.8 1.8-1.8"/></svg>

<!-- 메모와 함께 저장: 문서 + 여백 점(채움) -->
<svg viewBox="0 0 16 16"><path d="M4 1.5h5.5L13 5v9.5H4z"/><path d="M9.5 1.5V5H13"/><circle cx="10.5" cy="11" r="1.8" fill="currentColor" stroke="none"/></svg>

<!-- 내보내기: 문서 + 밖으로 나가는 화살표 -->
<svg viewBox="0 0 16 16"><path d="M12 9.5v5H4v-13h5"/><path d="M10.5 2h3.5v3.5M14 2 8.5 7.5"/></svg>

<!-- 연동: 순환 화살표 -->
<svg viewBox="0 0 16 16"><path d="M13.5 8a5.5 5.5 0 0 1-9.4 3.9M2.5 8a5.5 5.5 0 0 1 9.4-3.9"/><path d="M11.5 4.5h2.9V1.6M4.5 11.5H1.6v2.9"/></svg>
```

- 연동 버튼: 미결정 상태에서 우상단에 앰버 점(4px) 오버레이 — "누를 수 있음"의 조용한 신호.
- 연동 해제·복제는 아이콘 없이 컨텍스트 메뉴 텍스트만.

수용 기준: 4종 아이콘이 라이트 배경(`--chrome`)에서 `--muted`/`--ink` 색으로 렌더링 시 형태가 구분된다. T6가 이 스니펫을 그대로 복사해 쓸 수 있다.

## 3. D2 — 문구 정본

### 3.1 다운로드 분할 메뉴 (확정안)

| 액션 | 메뉴 문구 | 서브 텍스트 (있으면) |
|---|---|---|
| 파일만 | **PDF만 저장** | 파일 그대로 — 메모는 이 브라우저에 남습니다 |
| 메모포함 | **메모와 함께 저장** | 메모를 파일 안에 담습니다 |
| export | **내보내기 (다른 뷰어용)** | 하이라이트가 다른 PDF 앱에서도 보입니다 |

> 대안 검토: "사본 저장"(파일만) — 기각: 복제 액션과 혼동. "호환 저장"(export) — 기각: 무엇과 호환인지 불명.

### 3.2 연동 관련

- 연동 버튼 hover 툴팁: `'{대상 문서명}'과 연동` — **자동 선택된 대상을 실행 전에 노출**(§10 안내 2).
- 연동 성공 토스트: `'{대상 문서명}'과 연동됐습니다 — 메모가 함께 움직입니다.`
- 첫 오픈 힌트(문서당 1회, §10 안내 1): `'{원본 문서명}'의 사본입니다 — 메모를 같이 쓰려면 [연동]을 누르세요.`
- 연동 해제(컨텍스트 메뉴): `연동 해제` / 해제 토스트: `이제 이 문서의 메모는 따로 움직입니다.`

### 3.3 다이얼로그 (파괴적 동작에만)

**충돌** (§10 안내 3 — 단방향은 의도, 반대 방향 경로를 문구로 안내):
> 제목: **양쪽에 서로 다른 메모가 있습니다**
> 본문: '{대상}'과 이 문서가 각자 수정되었습니다. 지금 연동하면 **현재 문서의 메모로 덮어씁니다.**
> 다른 사본의 메모를 남기려면 취소하고, 그 문서를 열어 연동하세요.
> 버튼: `[현재 것으로 덮어쓰기]` `[취소]` — 기본 포커스 **취소**, Esc = 취소(미결정 유지)

**삭제 — 전용 통** (doc-identity-sync.md §5.1 경우 1):
> 이 문서를 삭제하면 **메모 {n}개가 함께 삭제됩니다.** `[삭제]` `[취소]` — 기본 포커스 취소

**삭제 — 공유 통** (§5.1 경우 2):
> 문서만 삭제됩니다 — **연동 중인 다른 사본의 메모는 유지됩니다.** `[삭제]` `[취소]`

**서명 PDF** (§6.1):
> 서명된 PDF입니다 — 메모를 파일에 담으면 **서명이 무효화됩니다.** `[계속 저장]` `[취소]`

### 3.4 스냅샷 안내 (§10 안내 4)

- 메모포함·export 저장 완료 토스트: `저장 시점의 메모가 담겼습니다 — 이후의 메모는 파일에 자동 반영되지 않아요.`
- 구식 자동 감지·재다운로드 유도 배지는 백로그(§11) — v1은 이 토스트 한 번이 전부.

수용 기준: 위 문구가 그대로 T6 코드에 들어간다(치환 변수는 `{…}` 표기). 모든 다이얼로그의 기본 포커스는 비파괴 버튼.

## 4. D3 — 플로우 명세

- **F1 첫 오픈 힌트**: 파생 노드(미결정)를 처음 열 때 1회. 노드에 `hintShownAt` 기록(스키마 T1에 필드 추가).
  토스트 5초, 액션 버튼 `[연동]` 포함(누르면 F2로). 이미 연동중/독립이면 표시 안 함.
- **F2 연동 실행**: 툴바 배지 또는 연동 버튼 클릭 →
  1. 대상 결정(정본 §5: {내 허브, 나를 허브로 둔 미결정 멤버} 중 최근 편집 1개). 대상 없으면 버튼 비활성 + 툴팁 "연동할 사본이 없습니다".
  2. 분기검사 무충돌 → **다이얼로그 없이 즉시 실행** + 성공 토스트. 충돌 → §3.3 충돌 다이얼로그.
  3. 취소 시 상태 변화 없음(미결정 유지).
- **F3 삭제**: 허브 카드 컨텍스트 메뉴 → §5.1 판정(전용/공유) → 해당 다이얼로그 → 실행. 삭제 토스트 없음(결과가 눈에 보임).
- **F4 다운로드 완료**: 파일만 → 토스트 없음(브라우저 다운로드 UI로 충분). 메모포함·export → §3.4 스냅샷 토스트 1회.
- 토스트 공통: 화면 하단 중앙, 기존 비PDF 안내 토스트 스타일 재사용. 동시 1개(교체).

수용 기준: T6가 이 문서만으로 추가 질문 없이 구현 가능. F1~F4 각각 시나리오 테스트가 [w33-qa.md](w33-qa.md)(D4, 추후)에 1:1로 대응.
